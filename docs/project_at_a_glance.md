# Project At A Glance

## Problem

Geopolitical risk analysis often begins with broad questions and fragmented evidence. Analysts need a repeatable way to classify scenarios, compare them with historical analogues, review available market evidence, and communicate caveats clearly.

## Workflow

```text
Question
  -> Scenario Classification
  -> Historical Analogue Retrieval
  -> Market Reaction Comparison
  -> Observed Pathways
  -> Executive Memo
  -> Markdown Export
```

## Data

- V1 historical analogue dataset focused on semiconductor-centered geopolitical competition.
- V2 US-China Strategic Competition pilot dataset.
- V2 market-reaction pilot dataset with selected historical return windows.
- Validation and coverage documentation that clearly states what is supported and what is not.

## Outputs

- Analyst Workbench Markdown report.
- Scenario taxonomy and question library.
- Historical analogue and market evidence documentation.
- Intelligence playbooks and analyst use cases.
- Executive demo and recruiter review guides.

## Business Value

- Saves first-pass research time.
- Standardizes scenario analysis.
- Makes evidence and limitations visible.
- Supports executive communication.
- Helps analysts avoid unsupported claims.

## Portfolio Value

The project demonstrates:

- business analytics product framing;
- risk analytics workflow design;
- data coverage auditing;
- schema-governed dataset expansion;
- market evidence methodology;
- responsible decision-support boundaries.

## Current Limitations

- V1 is validated on a narrow semiconductor-focused dataset.
- V2 market-reaction data is a static pilot and not fully integrated into the workbench.
- V2 coverage is not comprehensive.
- No live data, LLM APIs, forecasts, probabilities, expected returns, or investment advice are included.

## Future Expansion

The next expansion path is disciplined integration:

- V2 field transformation.
- V2 market-data validation.
- Additional US-China strategic competition coverage.
- Tests proving V2 integration does not break V1 behaviour.
