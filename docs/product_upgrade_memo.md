# Product Upgrade Memo

## Summary

The next portfolio upgrade is question-centric design.

The workflow remains unchanged:

```text
Question
  -> Classification
  -> Historical Analogues
  -> Market Reactions
  -> Observed Pathways
  -> Executive Memo
```

The product layer changes how the repository is explained: it is no longer only an event retrieval demo. It becomes a scenario intelligence workflow that starts from analyst questions, organizes them into scenario families, retrieves evidence, and packages current relevance without forecasting.

## Why Question-Centric Design Is Superior

Analysts do not start with event IDs. They start with questions:

- What happened after similar restrictions?
- Which sectors were affected historically?
- What analogues are relevant to this scenario?
- What evidence is strong enough for committee discussion?

Question-centric design matches analyst behavior. It makes the workflow easier to understand, easier to demonstrate, and easier to extend.

## Why Scenario Families Matter

Scenario families create reusable analytical frames.

They help analysts:

- classify ambiguous questions;
- compare current questions with historical analogues;
- separate technology competition from industrial policy or resource competition;
- identify relevant market proxies;
- disclose unsupported question boundaries.

Scenario families also help recruiters see product thinking rather than only code execution.

## Why This Improves Recruiter Value

The repo now demonstrates:

- analytics product framing;
- question-to-evidence workflow design;
- risk analytics taxonomy design;
- market evidence discipline;
- non-forecast decision support;
- analyst productivity orientation.

This is more compelling than a narrow event-centric script because it shows how an analytics product manager would structure a real analyst workflow.

## Why This Improves Analyst Usability

Question-centric design makes outputs easier to use because analysts can:

- start with natural questions;
- understand which scenario family applies;
- see historical analogues in context;
- review market evidence when available;
- interpret observed pathways;
- understand current relevance and limitations.

It also creates a better path for future dataset expansion because new events can be organized by scenario family.

## Why This Does Not Require Changing Workflow Logic

The existing deterministic workflow already starts with a question and ends with a memo. The upgrade is a product and documentation layer:

- scenario taxonomy defines how questions are grouped;
- question library provides reusable analyst prompts;
- current relevance framework explains interpretation;
- scenario intelligence layer describes how the same workflow should be understood.

No LLMs, APIs, dashboards, forecasts, probabilities, or investment recommendations are required.

## Recommendation

Keep the current workflow logic unchanged. Use the Scenario Intelligence Layer to present the repository as a question-centric geopolitical risk workflow platform supported by historical analogues and market evidence where available.
